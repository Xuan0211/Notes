Gradle更新失败（Error:Connection timed out: connect...）解决方法

一. 发现问题

当我们Copy一个项目或者在不同的机器打开AndroidStudio项目时，可能会遇到这样的问题：Gradle一直在转啊转，然后报错
Error:Connection timed out: connect. If you are behind an HTTP proxy, please configure the proxy settings either in IDE or Gradle.
大概翻译一下：错误：链接超时。如果你设置了HTTP代理，请检查设置。

二. 分析原因

  • 被墙导致相关Gradle使用包更新失败。
  • 设置了HTTP代理，去检查下有没有设置代理顺便查看是否配置成功。

三. 解决问题

这里只解决由于被墙而无法更新Gradle的问题。

3.1 查看当前项目Gradle版本

首先到项目的gradle-wrapper.properties文件里查看自己的Gradle包的版本，如下图我的就是gradle-3.3-all.zip
可以看到我这个项目的gradle包版本是gradle-3.3-all.zip。接下来找到自己的Gradle目录，默认是在C:\Users\你的用户名.gradle\wrapper\dists\gradle-3.3-all这个目录下。注意.gradle文件夹是隐藏的。打开这个目录会发现一个很多英文字母的目录，再打开就可以看到该项目的Gradle包了。如果发现这个gradle-3.3-all.zip的大小是0kb，说明是下载失败了。

3.2 替换本地Gradle包

到下方的链接找到刚才下载失败的Gradle包，然后到刚才的目录把下载好的zip文件替换掉。重启AS。
http://services.gradle.org/distributions/
Gradle Distributions

四. 额外小知识

另外也可以自定义gradle设置，File setting 搜索Gradle
第一个选项为使用默认的gradle wrapper：从网络下载并放到默认目录。第二个为使用本地gradle：选择下载好的到本地的目录。
