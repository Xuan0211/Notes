Chain Matrix Multiplication Problem 链矩阵乘法问题

Problem Statement

《算法竞赛入门经典》P277（电子书P429）
一个n×m矩阵由n行m列共个数排列而成。两个矩阵A和B可以相乘当且仅当A的列数等于B的行数。一个n×m的矩阵乘以一个m×p的矩阵等于一个的矩阵，运算量为mnp。 矩阵乘法不满足分配律，但满足结合律，因此A×B×C既可以按顺序(A×B)×C进行，也可以按A×(B×C)进行。假设A、B、C分别是2×3，3×4和4×5的，则(A×B)×C的运算量为2×3×4＋2×4×5＝64，A×(B×C)的运算量为3×4×5＋2×3×5＝90。显然第一种顺序节省运算量。
给出n个矩阵组成的序列，设计一种方法把它们依次乘起来，使得总的运算量尽量小。假设第i个矩阵Ai是pi−1×pip_{i-1}×p_ipi−1​×pi​ 的
Given a chain of [A1, A2, . . . , An] of n matrices for i = 1, 2, .. . , n, matrix Ai has dimension pi−1×pip_{i-1}×p_ipi−1​×pi​, find the order of multiplication which minimizes the number of scalar multiplications. 给定n个矩阵的 [A1，A2，.，An] 的链，对于i = 1,2，.，n，矩阵Ai具有维pi-1 x pi,找到最小化标量乘法次数的乘法顺序。

穷举法：

列举出所有可能的计算次序，并计算出每⼀种计算次序相应需要的数乘次数，从中找出⼀种数乘次数最少的计算次序

代码

int RecurMatrixChain(int i,int j)
{
    if(i==j) return 0;
    int minn=RecurMatrixChain(i,i)+RecurMatrixChain(i+1,j)+p[i-1]*p[i]*p[j];
    s[i][j]=i;//记录加括号的位置
    for(int k=i+1;k<j;k++) 
    {
        int t=RecurMatrixChain(i,k)+RecurMatrixChain(k+1,j)+p[i-1]*p[k]*p[j];
        if(t<minn)  
        {
            minn=t;
            s[i][j]=k;
        }
    }
    return minn;
}

备忘录方法：

使用备忘录：每个已经计算的子问题建立备忘，即保存子问题的计算结果以备需要时引用，以避免相同子问题的重复求解

代码：

int MatrixChain::LookupChain(int i, int j)
{
    if (m[i][j]>0) return m[i][j];  //子问题已解。
    if(i==j) return 0;                             
    int minn=LookupChain(i,i)+LookupChain(i+1,j)+p[i-1]*p[i]*p[j];   
    s[i][j]=i;//记录加括号的位置
    for (int k=i+1;k<j; k++) 
    {
         int t=LookupChain(i,k)+LookupChain(k+1,j)+p[i-1]*p[k]*p[j];
         if (t<minn) 
         {
              minn=t;
              s[i][j]=k;
         }
    }
    m[i][j]=minn;   
    return minn;                           
}

Dynamic Programming Solution 动态规划解决方案

Why

  • Problem is of type optimization 问题是类型优化问题
  • Optimal solution can be constructed 可以构建最优解
  • Sub-problems are dependant 子问题是相关的
  • Optimal structure can be characterized 可以表征最佳结构
  • Can be defined recursively 可以递归定义
  • Solution for base cases exits 基本情况的解决方案退出

上述方程有些特殊：记忆化搜索固然没问题，但如果要写成递推，无论按照i还是j的递增或递减顺序均不正确。正确的方法是按照j－i递增的顺序递推，因为长区间的值依赖于短区间的值

n<-length[p] – 1
for i <- 1 to n
    dp[i, i] <- 0

n<-length[p] – 1
for i <- 1 to n
    dp[i, i] <- 0
for l  <- 2 to n,//枚举区间的长度         
    for i <- 1 to n-l+1 //区间起始位置   
        j <- i+l-1//结束位置
        dp[i, j] <- ∞
        for k <- s[i,j-1] to s[i+1,j]//枚举断点
            t <- dp[i, k] + dp[k+1, j] + p[i-1]*p[k]*p[j]
            if i <= k && (k+1) <= j && t < dp[i, j]
               dp[i, j] = t
               s[i, j] <- k 
return m and s
