思路改进：BitMap

考虑到Hash导致的巨大额外空间开销，使用bitmap储存来节省

基本思想
使用bit位而实现Hash
适用于大规模数据，但数据状态又不是很多的情况。通常是用来判断某个数据存不存在的。
1个int占32位，那么我们只需要申请一个int数组长度为 int bitmap[1+N/32] 即可存储，其中N表示要存储的这些数中的最大值，于是乎：
bitmap[0]：可以表示0~31
bitmap[1]：可以表示32~63
bitmap[2]：可以表示64~95
……
如此一来，给定任意整数M，那么M/32就得到下标，M%32就知道它在此下标的哪个位置

给定任意整数MMM，显然有：
bitmap[M/32]=bitmap[M/32]∣(1<<(M%32))bitmap[M/32]=bitmap[M/32]|(1<<(M\%32))bitmap[M/32]=bitmap[M/32]∣(1<<(M%32))

若要查询M是否在bitmap中，那么：

性能分析
时间复杂度仍为O(n)O(n)O(n)但是常数略有变大
空间复杂度变为O(n/32)O(n/32)O(n/32)常数进一步减小

缺点分析
bitmap尽管压缩了空间，但是数据依然较为稀疏，仍有较大的空间浪费，这时可以考虑引入Roaring BitMap解决

#define N 1000
class BitMap
{
private:
    int bitmap[1 + N];
public:
    void init()
    {
        memset(bitmap, 0, sizeof(bitmap));
        return;
    }
    void add(int m)
    {
        bitmap[m / 32] = bitmap[m / 32] | (1 << (m % 32));
        return;
    }
    bool search(int m)
    {
        if (bitmap[m / 32] = bitmap[m / 32] & (1 << (m % 32)))
        {
            return true;
        }
        else
        {
            return false;
        }
    }
};
#define n 3
int float2int(float x)
{
    return (int)(x * pow(10, n));
}

main

    B.init();
    int S;
