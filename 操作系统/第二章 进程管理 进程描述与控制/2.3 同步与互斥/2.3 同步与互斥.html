2.3.1 同步与互斥的基本概念

1. 临界资源

一次仅允许一个进程使用的资源。
  • 物理设备：打印机
  • 变量、数据
do{
  entry section;    // 进入区：检查是否可进入，若是，设置访问标志
  critical section; // 临界区：进程访问临界资源的那段代码
  exit section;     // 退出区：将正在访问临界区的标志消除
  remainder section;// 剩余区：代码剩余部分
}while(true)

4. 一些准则

1）空闲让进。临界区空闲时，可以允许一个请求进入临界区的进程立即进入临界区。 2）忙则等待。当已有进程进入临界区时，其他试图进入临界区的进程必须等待。 3）有限等待。对请求访问的进程，应保证能在有限时间内进入临界区。 4）让权等待。当进程不能进入临界区时，应立即释放处理器，防止进程忙等待。
